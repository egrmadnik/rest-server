REST Server API Documentation

This server provides REST endpoints for managing users, devices, locations, and consumption data. All endpoints return JSON responses.

Base URL

http://localhost:3000

Available Resources

  • Users (/users)
  • Devices (/devices)
  • Locations (/locations)
  • Consumption (/consumption)

Example Requests

GET List All Resources

Retrieve all items for a resource.

# Get all users
curl http://localhost:3000/users

# Get all devices
curl http://localhost:3000/devices

# Get all locations
curl http://localhost:3000/locations

# Get all consumption data
curl http://localhost:3000/consumption

GET Get Single Item

Retrieve a specific item by ID.

# Get user with ID 1
curl http://localhost:3000/users/1

# Get device with ID 2
curl http://localhost:3000/devices/2

POST Create New Item

Create a new resource item. The name field is required.

# Create new user
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "John Doe", "email": "john@example.com"}'

# Create new device
curl -X POST http://localhost:3000/devices \
  -H "Content-Type: application/json" \
  -d '{"name": "Smart Thermostat", "type": "temperature"}'

# Create new location
curl -X POST http://localhost:3000/locations \
  -H "Content-Type: application/json" \
  -d '{"name": "Living Room", "floor": "1"}'

# Create new consumption record
curl -X POST http://localhost:3000/consumption \
  -H "Content-Type: application/json" \
  -d '{"name": "Daily Usage", "value": 45.2, "unit": "kWh"}'

PUT Update Item

Update an existing item by ID.

# Update user with ID 1
curl -X PUT http://localhost:3000/users/1 \
  -H "Content-Type: application/json" \
  -d '{"name": "John Smith", "email": "john.smith@example.com"}'

# Update device with ID 2
curl -X PUT http://localhost:3000/devices/2 \
  -H "Content-Type: application/json" \
  -d '{"name": "Smart Thermostat v2", "type": "temperature"}'

DELETE Delete Item

Delete an item by ID.

# Delete user with ID 1
curl -X DELETE http://localhost:3000/users/1

# Delete device with ID 2
curl -X DELETE http://localhost:3000/devices/2

Response Codes

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 400 Bad Request - Invalid request (e.g., missing required fields)
  • 404 Not Found - Resource not found
